# Docker Getting Started Guide

Това ръководство обяснява накратко какво е Docker, как се използва в рамките на този проект и какви са основните команди за управление.

## 1. Какво прави Docker?

Представи си го като инструмент за създаване на много леки, изолирани "виртуални машини", наречени **контейнери (containers)**. 

Вместо да инсталираш база данни (напр. PostgreSQL), Redis или сървър директно на компютъра си и потенциално да създадеш конфликти или "замърсяване" в системата си, Docker сваля готов *Image* (шаблон) и пуска сървиса изолирано в контейнер. 

*В случая с този проект, Supabase използва Docker, за да стартира локално цяла Postgres база данни, Auth сървър и Storage сървър само с 1 команда (`npx supabase start`), без да се налага да конфигурираш всичко това едно по едно.*

## 2. Управление на Docker на Системно Ниво

След като вече е инсталиран на твоя Linux (Ubuntu), Docker работи като системен процес (daemon) във фонов режим.

- **Провери дали Docker работи:**
  ```bash
  systemctl status docker
  ```
  *(Натисни `q`, за да излезеш от екрана със статуса)*

- **Спиране на Docker:**
  Ако искаш да пестиш RAM или батерия, когато не програмираш:
  ```bash
  sudo systemctl stop docker
  ```

- **Пускане на Docker:**
  Преди да стартираш отново локален сървър:
  ```bash
  sudo systemctl start docker
  ```

## 3. Как се управляват самите контейнери?

### Проверка на активните контейнери
За да видиш кои контейнери (виртуални машини) вървят в момента:
```bash
docker ps
```

### За проекти със Supabase (като текущия)
Supabase управлява контейнерите си автоматично. Използвай следните CLI команди:

- **Пускане / Стартиране на локалния бекенд:**
  Това сваля необходимите Docker images (ако ги няма) и пуска контейнерите.
  ```bash
  npx supabase start
  ```

- **Спиране на локалния бекенд:**
  Това стопира контейнерите на Supabase, освобождавайки ресурси на компютъра. Данните ти (базата) се запазват за следващия път.
  ```bash
  npx supabase stop
  ```

- **Премахване на всичко (Вкл. базата данни):**
  Ако искаш да започнеш напълно "на чисто" и да изтриеш локалната база данни:
  ```bash
  npx supabase stop --no-backup
  ```

### За други проекти (`docker-compose`)
Ако изтеглиш чужд проект и искат от теб да пуснеш бекенда, обикновено ще видиш файл `docker-compose.yml`. В тази папка просто пишеш:
```bash
docker compose up -d
```
*Това ще свали всички необходими images и ще пусне проекта във фонов режим (магически).*

За да го спреш:
```bash
docker compose down
```
